Table 表格 ​

基础用法 ​

配置columns表头和table-data表格数据。

数据多级显示 ​

columns中的 prop支持 x.y.z 多（无限）级数据形式。

提示

数据级不宜过多，可能会影响性能。

自定义状态小圆点的背景色 ​

配置项中valueType为select、radio或checkbox 时， columns中的options中可配置type或color属性。 color 优先级 高于 type。type支持'success' | 'warning' | 'info' | 'primary' | 'danger'。

自定义操作栏 ​

actionBar 的type 支持 button，icon和 link，对应 element plus 的 ElButton，ElIcon 和 ElLink 组件。

配置 confirm 即可实现二次确认。

配置 show 即可实现权限和动态显示。

拖动排序 ​

配置drag-sortable，同时配合dragSortEnd使用。

标题栏 ​

使用title，toolbar插槽。

分页栏 ​

配置pagination属性。

多选 ​

配置is-selection属性为true，即可显示表格多选框。默认值false。配合 el-table selection-change 事件使用。

树形结构 ​

配置tree-props树形为children: 'children'，数据结构中有 children 即可。 树形懒加载表格还需设置lazy，load。注意：children中的id和表格id不能重复。row-key默认值id。

展开行 ​

配置has-expand属性 true 可显示展开行，默认值 false。配合插槽 expand 使用。

可编辑表格 (整体) ​

可编辑表格 (单行) ​

可编辑和校验单行，也可以控制单行的每一个单元格的编辑状态和校验。核心方法是 调用 formRefs 里的startCellEdit()方法。

可编辑表格 (单元格) ​

可编辑和校验单元格。核心方法是 调用 formRefs 里的startCellEdit()方法。

自定义表格项和表单项 ( 插槽 ) 推荐 ​

提示

插槽 的优先级低于 render，高于数据。

PlusTable 组件会自动根据配置项的 prop 生成对应的插槽，例如下面的配置项，则会自动生成两个名称叫做 [ plus-cell-name ]和 [ plus-cell-status] 的两个插槽，插槽的生成规则就是 固定 key 值 [ plus-cell- ] 然后加上 配置项的 prop。

ts
import { PlusColumn } from 'plus-pro-components'

const tableConfig: PlusColumn[] = [
  {
    label: '名称',
    // 自动生成对应的插槽 'plus-cell-name'
    prop: 'name'
  },
  {
    label: '状态',
    // 自动生成对应的插槽 'plus-cell-status'
    prop: 'status'
  }
]

自定义表格项和表单项 (render) ​

提示

render 的优先级高于插槽。

  • 使用 render 函数自定义表格项。 render 方法可以返回以下三种情况

    • 返回一个 Component。 （配合PlusColumn 的 slots使用 ）推荐
    • 调用 vue 的渲染函数 h 返回一个 VNode。
    • 返回一个标签字符串 string。（渲染原生标签，配合PlusColumn 的 slots使用 ）

  • 自定义表格项还支持 renderHTML方法，需要返回一个 HTML 字符串。

  • 自定义表单项参考自定义表单项-renderField （ 表格中需要开启 editable）

自定义表格项和表单项 (jsx/tsx) ​

提示

jsx/tsx 的使用需要将 vue 单文件组件的script的属性 lang设置为jsx或者tsx，jsx 中值使用单花括号{}绑定。

html
<script lang="tsx" setup></script>
1

jsx/tsx 的支持本质是jsx/tsx解析数来是VNode, 使用 render 函数自定义表格项。

自定义表单项参考自定义表单项-jsx-tsx （ 表格中需要开启 editable）

自定义表格表头 (插槽) 推荐 ​

提示

插槽 的优先级低于 renderHeader，高于 label。

PlusTable 组件会自动根据配置项的 prop 生成对应的插槽，例如下面的配置项，则会自动生成两个名称叫做 [ plus-header-name ]和 [ plus-header-status] 的两个插槽，插槽的生成规则就是 固定 key 值 [ plus-header- ] 然后加上 配置项的 prop。

ts
import { PlusColumn } from 'plus-pro-components'

const tableConfig: PlusColumn[] = [
  {
    label: '名称',
    // 自动生成对应的插槽 'plus-header-name'
    prop: 'name'
  },
  {
    label: '状态',
    // 自动生成对应的插槽 'plus-header-status'
    prop: 'status'
  }
]

自定义表格表头 (renderHeader) ​

提示

renderHeader 的优先级高于插槽。

  • 使用 render 函数自定义表格项。 render 方法可以返回以下三种情况

    • 返回一个 Component。 （配合PlusColumn 的 headerSlots使用 ）推荐
    • 调用 vue 的渲染函数 h 返回一个 VNode。
    • 返回一个标签字符串 string。（渲染原生标签，配合PlusColumn 的 headerSlots使用 ）

自定义表格表头 (jsx/tsx) ​

提示

jsx/tsx 的使用需要将 vue 单文件组件的script的属性 lang设置为jsx或者tsx，jsx 中值使用单花括号{}绑定。

html
<script lang="tsx" setup></script>
